# Fangyu Geo

古代地缘形势推演工具。

Fangyu Geo 用地图、拓扑网络和指标图，把古代区域之间的地理屏障、交通通道、战略腹地、边防压力和进取路线组织成一个可探索的推演界面。它面向历史爱好者、地缘研究者和开发者，帮助用户更直观地理解一个地区为什么易守、为什么可攻、为什么会成为天下形势中的关键节点。

当前数据取材于《读史方舆纪要》的区域序文，并在此基础上整理为可视化所需的结构化节点、关系和判断。

## 项目功能

- 古代地缘形势推演：以拓扑网络展示核心地区、屏障、门户、通道、资源地和外部威胁之间的关系。
- 地理视图：在地形地图上查看关键地理节点，并辅助理解高低地势、山河阻隔和交通方向。
- 形势指标：对各区域的权重、守势、机动、风险进行可视化比较。
- 关系视图：把区域、主题和地点组织成关系图，方便观察论点之间的连接。
- 原文视图：保留文本入口，便于回看推演依据。
- 古今地名辅助：在节点信息和悬浮提示中显示古代地名大致对应的现代地区。
- 当前已为 15 个区域建立定制地缘拓扑模型：北直、南直、山东、山西、河南、陕西、四川、湖广、江西、浙江、福建、广东、广西、云南、贵州。

## 本地运行

安装依赖：

```bash

```

启动开发服务器：

```bash
npm run dev
```

然后在浏览器打开：

```text
http://localhost:3000
```

生产构建：

```bash
npm run build
```

代码检查：


## 数据说明

项目当前的数据层包括三类内容：

- 原始材料：`读史方舆纪要-各区域序.md`
- 区域摘要、指标和节点：`src/lib/fangyu-data.ts`
- 地缘拓扑模型：`src/lib/geostrategy-topology.ts`
- 古今地名映射：`src/lib/place-modern.ts`

目前这些内容以 TypeScript 对象形式维护，方便人工审阅、校订和版本管理。每一条摘要、指标、节点、边关系和战略判断，都应该可以继续追溯、讨论和修正。

## 后续 AI 接入方向

当前的结构化理解和拓扑模型是人工整理的第一版。后续可以引入 AI，把文本材料转成更稳定的结构化 JSON 数据。

计划方向：

1. 为章节理解和地缘战略拓扑定义稳定的数据结构。
2. 从原文中抽取地名、朝代、论点、证据和战略关系。
3. 每一条 AI 生成的判断都必须绑定原文证据。
4. 增加人工审核流程，把校订结果回写到数据层。
