# Fangyu Geo

《读史方舆纪要》历史地理与地缘战略可视化项目。

Fangyu Geo 试图把顾祖禹《读史方舆纪要》中的区域论述，从长文本阅读推进到可探索的知识界面。项目把原文、地理节点、战略指标、关系图和地缘战略拓扑推演放在同一个工作台里，方便研究者、历史爱好者和开发者继续校订、扩展与二次分析。

## 项目功能

- 读取并解析仓库内的 `读史方舆纪要-各区域序.md`
- 将 17 篇区域/专题序文拆分为结构化章节
- 为各区域提供可审计的结构化理解：
  - 核心论点
  - 战略姿态
  - 主题标签
  - 朝代线索
  - 地理节点
  - 权重、守势、机动、风险指标
- 提供多种可视化视图：
  - 地理图：展示各区域关键地理节点
  - 推演图：展示古代地缘战略拓扑关系
  - 关系图：展示章节、主题、地点之间的关系
  - 形势图：比较各区域战略指标
  - 原文：阅读并搜索原始文本
- 目前已为 15 个区域建立定制地缘战略拓扑模型：
  - 北直、南直、山东、山西、河南、陕西、四川、湖广、江西、浙江、福建、广东、广西、云南、贵州


```bash

```

```text
http://localhost:3000
```

生产构建：


代码检查：


## 数据说明

原始文本位于：

```text
读史方舆纪要-各区域序.md
```

当前结构化理解层位于：

```text
src/lib/fangyu-data.ts
src/lib/geostrategy-topology.ts
```

目前这些内容以 TypeScript 对象形式维护，方便人工审阅、校订和版本管理。每一条摘要、指标、节点、边关系和战略判断，都应该可以被追溯、讨论和修正。

## 后续 AI 接入方向

当前的结构化理解和拓扑模型是人工整理的第一版。后续可以引入 AI，把原文转成更稳定的结构化 JSON 数据。

计划方向：

1. 为章节理解和地缘战略拓扑定义稳定的数据结构。
2. 从原文中抽取地名、朝代、论点、证据和战略关系。
3. 每一条 AI 生成的判断都必须绑定原文证据。
4. 增加人工审核流程，把校订结果回写到数据层。
